What if a transformer kept thinking
before it spoke?

Looped transformers re-apply the same block of layers many times before producing a token. It is a way to spend more compute without spending more parameters - and a way to reason in latent space rather than out loud.

01

What is a looped transformer?

Intuition
tldr
A normal transformer stacks K distinct blocks. A looped transformer applies a shared block R times. Same parameters, more compute, and a residual stream that re-enters the same weights.

Picture the transformer as a tall building. In the ordinary version, every floor has its own interior and the elevator rides up once. A looped transformer is shorter, but the elevator goes through the same few floors several times before the doors open.

Concretely, a looped model has a short prelude, a shared recurrent block, and a short coda. The recurrence count can be fixed, sampled per sequence, or learned per token. Geiping '25Zeitoun '26

Go deeper · the recurrence as a mapmath · 2 min

Write the recurrent block as ht+1 = ftheta(ht, e). The current residual h and the original embedding e are mixed again and again. Parcae, Hyperloop, Huginn, and Ouro are mainly different ways of making that repeated map useful and stable.

Three things to keep in your head

  1. Parameters and compute decouple. One shared block times many iterations means more FLOPs without more memory.
  2. The residual stream is a state. Looping turns a transformer into a discrete dynamical system.
  3. R is a knob. You can spend it at train time, test time, or per token.
02

Why loop?

Motivation
tldr
Three reasons converge: parameter efficiency, latent reasoning, and elastic test-time compute.

1 · Parameter efficiency

Edge deployment is memory-bound. Hyperloop gets depth-matched behavior with roughly 50% fewer parameters by reusing the middle block.

2 · Latent reasoning

Chain-of-thought turns hidden state into text and back. Looped models keep iterating in continuous latent space, preserving information that is hard to express as a word.

3 · Elastic test-time compute

With a loop count or learned exit, the model can spend extra compute only where it pays off: hard tokens, hard questions, or slow-to-converge residual states.

↻
The unifying insight. Looping makes inference depth separable from parameter count.

04

Mechanism: fixed points & cyclic trajectories

New finding
tldr
When looped models train well, layers inside the recurrent block often converge to distinct fixed points. The block as a whole traces a stable latent cycle.

This is the most intuition-shifting result: the model runs a stable iterative computation over the same residual stream.

Once attention patterns stabilize, extra recurrence stops being chaotic and starts acting like sustained computation. That is why some models extrapolate beyond the depth seen in training.

"Each layer in the cycle converges to a distinct fixed point; the recurrent block follows a consistent cyclic trajectory."Blayney et al. 2025

Three regimes seen in practice

  • Fixed point. States stop moving. This is stable and extrapolates well.
  • Orbit. States cycle through a closed loop.
  • Slider. The state drifts monotonically along a latent direction.
05

Stages of inference, repeated

tldr
Looped transformers learn to repeat the four standard stages of inference once per recurrence.

Feedforward transformers move through detokenization, feature build-up, reasoning, and token prediction. Looped models compress those phases into the recurrent block and replay them on a refined hidden state.

This makes the recurrent block a miniature transformer whose input is the prior pass's residual stream.

★
Interpretability consequence. If the stages repeat per loop, feedforward mech-interp tools can be applied per recurrence.
06

Training stability & scaling laws

Updated 2026-04
tldr
The classic failure mode is residual explosion. Parcae diagnoses it as a spectral-norm problem and fixes it structurally.

Early looped recipes suffered loss spikes and recurrent-state norms swinging across orders of magnitude. Parcae traces the failure to repeated application of an unstable linearized map, then bounds it with a negative-diagonal parameterization.

Go deeper · per-sequence samplingtraining · 2 min

Instead of sampling one recurrence count for a whole batch, Parcae samples per sequence and pads to the batch max. Shorter sequences hold state, producing a better estimate of steady-state behavior.

Scaling laws so far

  • FLOPs scaling. Loops and data must increase together.
  • Test-time scaling. Extra loops follow a saturating exponential, with a task-specific compute appetite.
07

Test-time compute scaling

tldr
A trained looped model can often be run at larger R during inference. Quality climbs until the task saturates.

The scrubber on the right shows the shape: reasoning-heavy tasks keep improving with more recurrences, while knowledge-heavy tasks saturate early.

  • Zero-shot adaptive compute. Stop when successive residual states stop changing.
  • Per-token early exit. A learned exit head can choose the loop budget token by token.
08

Latent reasoning vs chain-of-thought

Walkthrough
tldr
Both models eventually produce one token. The difference is what happens between the prompt and that token.

The figure on the right is an eight-step walkthrough: same question, same prompt, same final token, and two different paths through the model.

The key asymmetry: bandwidth

A CoT model must pass every thought through the vocabulary. A looped model keeps the thought in the residual stream and updates that continuous vector directly.

"One token at the end is not the point; the state feeding the unembed carries more worked-through thought."The essential asymmetry

11

Implications

For deployment

Small looped models can trade extra inference compute for lower memory footprint, which matters for phones, browsers, and edge hardware.

For reasoning

Reasoning becomes a continuous dial rather than a fixed depth or a visible chain of generated tokens.

For interpretability

Stable cycles and fixed points give probes a repeated structure to inspect.

12

Glossary

Looped TransformerA transformer that re-applies the same block of layers multiple times.
Residual streamThe running vector each block reads from and writes to.
RecurrenceThe number of times the shared block is applied.
Fixed pointA state where applying the function leaves the state unchanged.
Ponder gateA learned exit head that predicts when to stop looping.
Hyper-connectionA matrix-valued residual stream used to increase loop capacity.
