What the hidden state does when the block repeats

The mechanistic paper looks inside looped reasoning models after the architecture is already trained. It asks a direct question: as the same block runs again and again, do the internal states settle, cycle, drift, or keep changing?

Object measured
residual state

Same layer, same token, compared across recurrence steps.

01

The measurement

Setup
paper setup
Take the residual stream after each layer in the recurrent block. Run many recurrences. Compare each layer's state with the same layer one loop later.

The paper studies cyclic recurrent blocks in models such as Huginn-0125, Ouro 1.4B, and retrofitted Llama-style models. For each token, recurrence creates a realized depth: layer 1, layer 2, layer 3, layer 4, then layer 1 again on the next pass.

The useful comparison is layer-to-layer across cycles. If the output of layer 2 at recurrence 18 looks almost the same as layer 2 at recurrence 19, layer 2 has essentially settled. If every layer settles to its own endpoint, the block as a whole follows a repeated cycle through those layer endpoints.

01
Collect states by realized depth.

The same physical block appears many times in the unrolled computation. The paper indexes the residual state by recurrence and by layer inside the block.

02
Compare a layer with itself one loop later.

The question is whether layer 3 at pass 20 has become close to layer 3 at pass 21. That avoids mixing together different layers in the cycle.

03
Check the attention pattern too.

Converged residuals are stronger evidence when attention matrices also stop changing. The paper uses both views to separate stable recurrence from visual coincidence.

Why the word “cyclic” matters

A recurrent block can reach a stable behavior without every intermediate layer becoming identical. Layer 1 can settle near one state, layer 2 near another, and so on. The sequence of layers still moves through a cycle.

That is the pattern the paper emphasizes: layer-specific fixed points inside a repeating block.

02

Huginn settles quickly

Observed
paper reading
Huginn's residual states approach layer-specific references early enough that additional recurrence stays in-distribution.

Huginn-0125 shows fast convergence in the fixed-point plots. After the early recurrences, the difference between a layer's current state and its late-recursion reference state becomes small. Attention patterns also stabilize.

This is a good fit for test-time recurrence. If the hidden state has a stable limiting behavior, running additional loops is less likely to push the model into a strange region of activation space. Extra recurrence can refine the same computation rather than sending the state somewhere unrelated.

  • Residual states: Huginn moves toward stable layer-wise references.
  • Attention: attention matrices show the same recurring structure after the transient phase.
  • Test-time depth: this helps explain why Huginn can run beyond the recurrence counts used during training.
03

Ouro is different

Important detail
paper reading
Ouro has recurrence structure, but the strict fixed-point claim does not fit the 1.4B model's long-recursion behavior.

Ouro 1.4B does show cyclic similarity. The same layer across neighboring recurrences can look related. The fixed-point test is stricter, and Ouro does worse there: its states keep changing across long recurrences.

This distinction matters. A page that says “looped models converge to fixed points” is too broad. The paper's actual picture is more interesting: some models settle into fixed points, some have stable cycles or drifting behavior, and Ouro is one of the cases where strict fixed-point language is too strong.

04

Orbits and sliders

taxonomy
The paper's categories give you three different stable-looking behaviors to check: fixed points, bounded orbits, and directional sliders.

Geiping et al. had already observed latent orbits and sliders in Huginn. The mechanistic paper follows up on that behavior. An orbit is a bounded repeated movement. A slider is a directional drift. Both can appear alongside fixed-point behavior in intermediate layers.

The practical reading is simple: stable looped inference does not always mean the hidden state freezes. Bounded repeated motion can also be stable. A drifting direction can also carry useful state, especially if the model uses it like a counter.

05

Stages of inference

separate measurement
Stage structure and fixed-point structure are related evidence, but the paper measures them separately.

The paper also measures stage-like behavior across the repeated block. Ouro and retrofitted models show stages that resemble the usual feedforward pattern. Huginn is a negative case here: it does not show the same clean stage structure, likely due to how its normalization controls activation magnitude.

This gives a cleaner way to talk about the field. Fixed-point behavior and stage-of-inference behavior are related, but they are separate measurements. A model can show one pattern more clearly than the other.

06

How to use this idea

practical check
Extra recurrence is safest when residual states, attention patterns, and performance all remain stable beyond the training range.

When evaluating a looped model, ask three questions before trusting extra recurrence:

  1. Do same-layer residual states approach stable references?
  2. Do attention patterns stabilize across cycles?
  3. Does performance hold when recurrence is pushed past the training range?

If all three hold, recurrence is behaving like a reliable compute budget. If one fails, the model may still work, but the explanation needs more care.
